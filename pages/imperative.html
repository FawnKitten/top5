Imperative programming

Imperative programming is the oldest style of programming, it consists of giving the computer step by step instructions, instead of telling the computer what to do (declarative programming), telling the computer how to do it

This style of programming is the oldest because when computer technology was first being invented by Alan Turing, there were no compilers/interpreters/assemblers, so they had to give instructions directly to the computer in binary form

Although other styles emerged (Object Oriented, Functional, Data Oriented) this style is still commonly used, especially when programming embedded devices

Disadvantages

  • this style is less maintainable because code is not very organized
  • this style is harder to debug because variables are always changing and
