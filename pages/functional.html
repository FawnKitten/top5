Functional Programming

Functional programming is all about pure functions. Pure functions that take only an input and an output, and only that, there is no change of state

Terms generally used in FP

  • Monad -
    A function whos return type holds a type.

  • Recursion -
    A funciton that calls it's self, either directly or indirectly.

  • higher order function -
    A function that returns, or takes as an input another funcion.

  • referential transparency -
    all variables are immutible during the duration of a program
